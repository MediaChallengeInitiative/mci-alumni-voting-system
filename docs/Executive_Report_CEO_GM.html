3. SYSTEM CAPABILITIES

3.1 Automated Voter ID Generation

Format: MCIA + First Initial + First 2 Letters of Lastname + 25

Example: Emmanuel Bahindi → MCIAEBA25

Features:

  • Unique IDs generated automatically
  • No manual ID assignment required
  • Collision handling with numeric suffixes
  • All uppercase for consistency

3.2 Security Framework - 5 Layers of Protection

Layer 1: Voter ID + Password Authentication
Layer 2: Device Fingerprint Binding
Layer 3: One-Time Vote Enforcement
Layer 4: Session Token Validation
Layer 5: Automatic Post-Vote Logout

3.3 Fraud Prevention Matrix

Threat Prevention Mechanism
Multiple voting attempts Vote flag + automatic logout after submission
Device sharing between voters One device = One voter binding
Credential sharing Device fingerprint verification
Session hijacking Unique token validation per request
Unauthorized database access Password hashing (bcrypt, cost 12)
